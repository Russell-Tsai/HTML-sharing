Root Cause Analysis
P0 Stevia Layout — v10.24.0 regression
151 total events, 142 users. Three crashes (INLINE-60301, 60307, 60418) all first appeared Jun 12 — four days after v10.24.0 shipped. All crash in Stevia+Hierarchy.swift UIView.subviews().

v10.24.0 shipped two separate PR iterations for fix/reservation-form-bring-front (PR #10766) and fix/rsv-form-bring-front (PR #10786), which "adds back original logic but only when reservationView doesn't exist." This two-step patch suggests the first fix was incomplete — the code is likely accessing a deallocated view during transition, causing Stevia's layout pass to SIGSEGV.
P0 Timeline Dragging — v10.22.0/v10.23.0
INLINE-60018 (10 events) appeared Jun 5, after v10.23.0. Crashes at TimelineViewController+Dragging.m:108 in collectionViewDidLongPress:.

v10.22.0 shipped heavy PDA drag changes: alert on drag, disallow swap, tooltip flip. v10.23.0 followed with another fix for PDA swap alert. These layered changes likely left a state inconsistency in the long-press drag initiation path.
P1 CoreData Observer — v10.22.0
INLINE-59751 (48 events, 45 users) appeared May 27 — two days after v10.22.0. Crashes in FetchedResultsControllerEntityObserver<CDTakeout>.

v10.22.0 bumped OmniChannel to v4.1.1 which may have changed how CoreData FRC observers are set up for Takeout entities. The CDTakeout observer is being accessed after deallocation or before setup completes.
iOS 26 UIKit Init / NSMallocException
Multiple crashes starting May 22 — before any of the three releases — at low-level UIKit init paths: UILabel.init, UITextField.init, UITextView.init, UICollectionView.init, plus NSMallocException in UIStackView and UICollectionView+Reusable.

These signatures (SIGSEGV/NSMallocException in UIKit init, 1 event/1 user pattern) are consistent with iOS 26 beta changing UIKit internal memory layout. Not app-code regressions — check if Stevia has an iOS 26 update too.
